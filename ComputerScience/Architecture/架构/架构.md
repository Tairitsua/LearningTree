# 架构

本目录将 [架构.md](../架构.md) 拆分为一套更适合长期维护的架构笔记。整理时尽量保留原始知识点，只修复标题层级、列表格式、英文术语、代码块与少量明显的事实或拼写问题。

## 阅读顺序

1. 先读 [架构基础与设计原则](架构基础与设计原则.md)，建立企业架构、设计原则、`IoC`、`DI`、`AOP` 等基础概念。
2. 再读 [微服务与分布式架构](微服务与分布式架构.md)，理解 `SOA`、微服务、网关、服务治理与三层架构。
3. 然后读 [DDD基础](DDD基础.md) 与 [限界上下文与领域模型](限界上下文与领域模型.md)，把 `DDD`、上下文映射、实体、值对象、聚合等主线梳理清楚。
4. 最后读 [架构模式与UML](架构模式与UML.md)、[设计模式](设计模式.md) 与 [交付流程与软件工程](交付流程与软件工程.md)，补齐架构图、模式与工程实践。

2. 章节按“基础概念 -> 架构风格 -> 领域建模 -> 建模表达 -> 工程实践”组织。
