# 基础知识

## 常见术语

1. `CMS` 是 `Content Management System` 的缩写，意为“内容管理系统”。
2. `HTTP` 是 `HyperText Transfer Protocol`，即超文本传输协议。
3. `FTP` 是 `File Transfer Protocol`，是 `TCP/IP` 协议族中的常见文件传输协议。
4. `FTP` 默认使用 `TCP` 的 `20` 和 `21` 端口。
5. `WebShell` 是部署在服务器上的后门脚本，可基于 `ASP`、`PHP`、`JSP` 等环境执行。

`FTP` 由 `FTP` 服务器与 `FTP` 客户端组成。开发网站时，经常利用 `FTP` 将网页或程序上传到 `Web` 服务器。由于传输大文件效率较高，它也常被用于批量文件传输。

## `file` 协议

`file` 协议主要用于访问本地计算机中的文件，效果类似于在 `Windows` 资源管理器中直接打开文件。

基本格式如下：

```txt
file:///文件路径
file:///f:/flash/1.swf
```

之所以是三个 `/`，是因为 `URL` 的基本结构是：

```txt
schema://host[:port]/path?[query-string]#anchor
```

`file` 协议通常没有 `host`，因此会表现为 `file:///`。

## `URI` 与 `URL`

1. `URI` = `Uniform Resource Identifier`，统一资源标识符。
2. `URL` = `Uniform Resource Locator`，统一资源定位符。
3. `URN` = `Uniform Resource Name`，统一资源名称。

`URI` 是更大的概念，`URL` 与 `URN` 都属于 `URI`。在日常 `Web` 场景里，最常见的仍然是 `URL`。

`URL` 的常见结构如下：

```txt
schema://host[:port]/path?[query-string]#anchor
```

示例：

```txt
http://dun.163.com/sj/test/test.jsp?name=sviergn&x=true#stuff
```

## `HTTP` 基础

1. `Referer` 请求头会告知服务器该请求的来源，常用于防盗链、来源校验等场景。
2. `Set-Cookie` 响应头用于由 `Web` 服务器向浏览器下发凭证或状态。
3. `HTTP` 常见请求方法有 `GET` 和 `POST`。
4. `301` 与 `302` 状态码都表示跳转，但语义不同。

`GET` 常用于从指定资源请求数据，参数通常出现在 `URL` 中。`POST` 常用于向指定资源提交数据，参数通常放在请求体中。

## 浏览器与页面对象模型

1. `DOM` 是 `Document Object Model`，即文档对象模型，用于把 `HTML`、`XML` 文档表示为可编程对象结构。
2. `BOM` 是 `Browser Object Model`，即浏览器对象模型，用于和浏览器窗口本身交互。

参考链接：

## `Web` 服务端架构

`Web` 服务端通常由以下几部分组成：

1. `Web` 服务器
2. 语言解释器或运行时
3. 数据库

常见组合如下：

```txt
Windows Server: IIS + ASP(.NET) + SQL Server
Linux: Apache + PHP + MySQL
UNIX/Windows: Tomcat + JSP + Oracle
```

这些并不是固定搭配，只是常见方案。实际环境中可以灵活组合。很多时候也可以通过首页脚本后缀判断技术栈，例如 `.asp`、`.aspx`、`.jsp`、`.php`。

## 其他常见知识

### `SQL` 与 `HTML`

1. `SQL` 关键字通常对大小写不敏感。
2. 语句一般以分号结尾，很多环境也会自动补全。
3. `HTML` 标签在 `HTML` 语法中通常不区分大小写。

创建数据库示例：

```sql
CREATE DATABASE websecurity;
```

如果 `HTML` 页面出现乱码，可以在文档头中增加：

```html
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
```

### 搜索引擎语法

1. `intitle:keyword` 表示标题中包含指定关键词。
2. `intext:keyword` 表示正文中包含指定关键词。
3. `site:domain` 表示仅在某个域名及其子域名范围内搜索。
4. `filetype:pdf` 等可限定文档格式。

示例：

```txt
photoshop 实用技巧 filetype:doc
```

```txt
IPv4 回环地址: 127.0.0.1
IPv6 回环地址: ::1
主机名: localhost
```
